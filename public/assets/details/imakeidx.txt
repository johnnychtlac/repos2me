imakeidx 是一个用于 LaTeX 的宏包，旨在简化索引（index）的创建和管理过程。它的主要功能是利用现代 TeX 发行版（如 pdfTeX, XeTeX, LuaTeX）的 \write18 功能（即 shell escape），在编译文档的过程中自动调用外部程序（如 makeindex 或 xindy）来排序和格式化索引条目。这意味着用户通常只需编译一次文档（在启用 shell escape 的情况下），即可生成包含已排序索引的最终 PDF，而无需手动在终端运行索引程序或进行多次编译循环。在排版过程中自动运行索引排序程序（默认为 makeindex，也可配置为 xindy）。支持自动分割原始索引文件、排序并格式化，然后将结果直接嵌入文档中。允许在一个文档中轻松创建多个不同的索引（例如：人名索引、主题索引、代码索引等）。通过 \makeindex[name=..., title=...] 定义多个索引，并使用 \index[name]{entry} 将条目归类到特定索引。可通过选项 makeindex (默认) 或 xindy/texindy 指定排序引擎。提供 splitindex 选项，利用 Markus Kohm 的 splitindex 脚本将单个原始索引文件拆分为多个部分，解决了 LaTeX 对同时打开文件数量的限制，特别适合需要大量索引的情况。支持自定义索引的列数 (columns)、列间距 (columnsep)、是否添加列间规则 (columnseprule) 等。通过 intoc 选项可将索引标题自动添加到文档目录中。提供 \indexsetup 命令来自定义索引的章节级别、页眉、页面样式等。文档详细讨论了与 memoir 类、showidx、fancyhdr、combine、ledmac 等包或类的兼容性问题及解决方案。特别指出若使用 twocolumn 选项或 original 选项，某些自定义布局设置将被忽略，以保留类文件的原始行为。提供了当索引未正确生成时的排查指南，例如检查是否启用了 -shell-escape (或 MiKTeX 下的 --enable-write18)，检查日志文件中的 runsystem(...)...executed 记录，以及检查生成文件的时间戳。需要使用支持 shell escape 的编译器命令，例如：pdflatex -shell-escape filename.tex（MiKTeX 用户可能需使用 --enable-write18）。
