algxpar 是基于 algorithmicx/algpseudocode 的扩展包，它的核心作用是完美处理多行文本的缩进与排版，并提供了丰富的自定义选项，特别适合教学场景中编写描述性更强、更易读的算法。在教学中，算法描述往往偏向自然语言（如“将当前费用累加到总成本中”），而非简短的数学符号（如 s=s+c ）。标准包遇到换行时容易破坏缩进结构。提供了 \Statep 命令，能够自动处理跨行文本，确保后续行与基线正确对齐，保持视觉清晰度。支持行内注释（\CommentIn{...}，使用 ▷ 和 ◁ 包裹）、局部注释（\Commentl{...}）和传统的右侧注释。注释语法更友好，支持在块结束处（如 \EndWhile）直接添加注释。支持标准的 if、switch、for、while、repeat-until 以及 loop 等结构。提供了 \Procedure 和 \Function 用于模块化编程。
\Statep：核心命令。替代标准的 \State，支持多行文本自动缩进。编写包含长描述的算法步骤。
\CommentIn：内联注释。注释符号出现在行内文本的中间，而非行尾。需要在代码逻辑中间插入解释时。
\Id{}：标识符命令。用于变量名（如 sum），自动处理字体并在数学模式下防断词。强调变量名，使其与正文区分开。
\AlgSet：全局或局部设置。用于修改算法的样式（颜色、字体、缩进线等）。统一整篇文档或单个算法的视觉风格。
\Description：算法描述。用于定义算法的标题、输入、输出等文档信息。算法的前置说明部分。
algxpar 提供了非常精细的控制选项（Options），允许用户通过 \AlgSet 或环境选项来调整外观：实验性功能，支持显示垂直的缩进引导线，帮助读者追踪代码块的层级（类似 Python 的缩进视觉效果）。可通过 line style 选项自定义线条颜色、虚实和箭头。可独立设置关键词（Keywords）、注释（Comments）、文本（Text）和常量（True/False）的颜色。可修改关键词、常量、模块名（Procedures）的字体族、粗细和大小。内置支持英语（English）和巴西葡萄牙语（Brazilian）。用户可以通过 \AlgLanguageSet 定义自己的语言包（如将 "if" 改为 "如果"）。通过 noend 选项，可以隐藏 end if、end while 等结束行，仅保留缩进，使代码看起来更像 Python 或现代编程语言。
