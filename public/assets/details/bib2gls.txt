bib2gls 的核心作用是将传统的 .bib 参考文献数据库文件转化为 LaTeX 的词汇表（Glossary）、缩略语表（Abbreviations）和索引（Index）定义，从而极大地简化了复杂文档的术语管理。通常使用 glossaries 宏包时，需要在 .tex 文件中用 \newglossaryentry 等命令定义术语。当术语量巨大时，文档导言区会变得臃肿且难以维护。bib2gls 方案允许你将所有术语定义存储在 .bib 文件中（类似管理参考文献）。你可以使用 JabRef 等参考文献管理软件来维护这些术语，实现了内容与格式的分离。它不仅能提取定义，还能根据文档中实际使用的术语进行筛选。通过 record 选项，它只提取文档中实际引用的条目（以及它们的依赖项），从而减少不必要的定义，优化编译资源。bib2gls 内置了强大的排序功能。它可以直接读取 .aux 文件中的位置信息，对术语进行排序并合并位置列表（Location Lists）。这意味着你通常不再需要调用 makeindex 或 xindy 来生成词汇表。支持基于 Java 的 Locale（如 en-GB, de-DE 等）进行复杂的语言敏感排序（包括重音、大小写处理等）。
.bib 支持条目类型如 @entry，@symbol，@abbreviation，@index 等。输出格式 .glstex 文件。包含 LaTeX 可读的 \bibglsnewentry 等定义命令。数据处理支持层级结构（父子条目）、跨资源引用、条件定义和复杂的字段处理（如大小写转换、字段拼接）。构建流程 pdflatex -> bib2gls -> pdflatex（-> pdflatex 如果需要更新引用）。
你在 .tex 中使用 \gls{bird}，LaTeX 会将这个引用记录在 .aux 文件中。bib2gls 读取 .aux 文件，知道哪些条目被使用了；同时读取 .bib 文件获取条目的完整定义；然后根据规则进行排序、分组、处理依赖关系。bib2gls 将处理好的条目定义写入 .glstex 文件。下一次 LaTeX 编译时，通过 \GlsXtrLoadResources 读取 .glstex 文件，此时所有需要的术语命令都被定义好了，可以正确显示在文档和目录中。至少需要 Java 8 运行时环境（JRE）。必须配合 glossaries-extra 宏包（至少版本 1.12）使用，不能单独与基础 glossaries 宏包配合。文档中还提到了配套的转换工具 convertgls2bib（用于将旧的 .tex 术语定义转换为 .bib 格式）和 datatool2bib。
